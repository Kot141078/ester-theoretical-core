# Contributing and public review

Use GitHub issues for reproducibility reports, conceptual reviews, or proposed
counterexamples. Pull requests are welcome for clearly scoped corrections,
additional tests, documentation, or independent analysis implementations.

A strong issue contains:

```text
release tag and SHA-256
platform/dependency versions
exact file and line range
minimal scenario
observed result
expected result or challenged claim
conceptual vs reproduced status
acceptance criterion
```

Do not submit private memory, dialogue, credentials, personal data, or live
system records. The public repository is an architecture and review surface,
not a data dump.
